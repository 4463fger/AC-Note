# 排序算法总结

---
## 稳定性
>排序算法的稳定性是指：同样大小的样本在排序之后会不会改变原始的相对次序
---
### **总结表**
| 算法                | 时间复杂度(平均) | 空间复杂度   | 稳定性 |
|-------------------|-----------|---------|-----|
| **SelectionSort** | O(N²)     | O(1)    | 无   |
| **BubbleSort**    | O(N²)     | O(1)    | 有   |
| **InsertionSort** | O(N²)     | O(1)    | 有   |
| **MergeSort**     | O(N*logN) | O(N)    | 有   |
| **QuickSort**     | O(N*logN) | O(logN) | 无   |
| **HeapSory**      | O(N*logN) | O(1)    | 无   |
| **CountSort**     | O(N)      | O(M)    | 有   |
| **RadixSort**     | O(N)      | O(M)    | 有   |
---

> 在数据量非常小的情况下可以做到非常迅速：插入排序
> 
> 性能优异、实现简单且利于改进(面向不同业务可以选择不同划分策略)、不在乎稳定性：随机快排
> 
> 性能优异、不在乎额外空间占用、具有稳定性：归并排序
> 
> 性能优异、额外空间占用要求O(1)、不在乎稳定性：堆排序
